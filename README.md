## Overview

Customer churn is one of the most costly problems in retail banking. This project builds a **binary classification pipeline** to identify at-risk customers before they leave, enabling the business to intervene with targeted retention campaigns.

**Target variable:** `Exited` — `1` = churned, `0` = stayed

---

## Dataset

| Property | Detail |
|---|---|
| Source | Internal bank data (anonymised) |
| Format | Excel (.xlsx) — 2 sheets |
| Rows | ~10,000 customers |
| Features | 12 raw → 19 after engineering |
| Class balance | 79.6% stayed / 20.4% churned |

**Sheets:**
- `Customer_Info` — CreditScore, Geography, Gender, Age, Tenure, EstimatedSalary
- `Account_Info` — Balance, NumOfProducts, HasCrCard, IsActiveMember, **Exited**

## Results

| Model | Test AUC-ROC | CV AUC (5-fold) | Accuracy |
|---|---|---|---|
| Logistic Regression | 0.790 | 0.784 ± 0.013 | 74% |
| Random Forest | 0.859 | 0.855 ± 0.011 | 82% |
| **Gradient Boosting ** | **0.868** | **0.864 ± 0.009** | **86%** |

> **Gradient Boosting** was selected as the best model. The near-identical Test and CV AUC scores confirm the model generalises well and is not overfitting.

## Key Findings

### From EDA
- **Germany** has a churn rate of **32.5%** — nearly double France (16.2%) and Spain (16.7%)
- **Female** customers churn at **25.1%** vs 16.5% for males
- Customers with **3–4 products churn at 83–100%** — a critical red flag
- **Inactive members** churn at **26.9%** vs 14.3% for active members

### From the Model
- **Age** is the single most important predictor of churn
- **NumOfProducts**, **IsActiveMember**, and **Balance** are the next strongest signals
- The engineered **BalanceSalaryRatio** feature ranked in the top 15

### Business Recommendations
-  Target retention campaigns at **older, German, female, and inactive** customers
-  Investigate the **multi-product experience** — 3+ products is a churn signal, not a loyalty one
-  Launch a **re-engagement programme** for inactive members
-  Commission a **Germany-specific review** of service quality and competitive landscape

## How to Run

### 1. Clone the repo
```bash
git clone https://github.com/your-username/bank-churn-prediction.git
cd bank-churn-prediction
```

### 2. Install dependencies
```bash
pip install pandas numpy matplotlib seaborn scikit-learn openpyxl
```

### 3a. Run the full notebook (recommended)
```bash
jupyter notebook bank_churn_notebook.ipynb
```

### 3b. Or run scripts step by step
```bash
python step1_data_cleaning.py    # generates bank_churn_clean.csv
python step2_eda.py              # generates 4 EDA charts
python step3_modelling.py        # generates 3 model evaluation charts
```
